# Contributing

Danke, dass du beitragen willst!

Kurzer Leitfaden:

- Forke das Repo oder arbeite auf einem Feature‑Branch: `feature/<kurz-beschreibung>`
- Schreibe Tests für neue Funktionen.
- Formatiere Code und halte dich an bestehende Konventionen.
- Eröffne einen Pull Request (PR) mit einer kurzen Beschreibung.
- Mindestens ein Reviewer sollte den PR akzeptieren.

Commit‑Nachrichten: kurze Typen wie `feat:`, `fix:`, `chore:`.

Bei Fragen, schreibe im Projekt‑Chat oder eröffne ein Issue.
